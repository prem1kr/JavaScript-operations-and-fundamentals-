Q2. Describe the categorization of operators in JavaScript based on their functionality. Provide examples for each category.

Types of Operators in JavaScript:

1. Arithmetic Ooprators:- Used for mathematical calculations.
  • Examples: + (addition), - (subtraction), * (multiplication), / (division), % (modulus), ++ (increment), -- (decrement).
  • Example: let sum = 10 + 5; (The + operator adds 10 and 5.)

    • 2. Comparison Operators:- Used to compare two values.
  • Examples: == (equal to), != (not equal to), === (strict equal), !== (strict not equal), > (greater than), < (less than).
  • Example: if (age > 18) (The > operator checks if age is greater than 18.)

    • 3. Logical Operators:- Used to combine or invert boolean values.
  • Examples: && (logical AND), || (logical OR), ! (logical NOT).
  • Example: if (isAdult && hasTicket) (The && operator checks if both conditions are true.)

    • 4. Assignment Operators:- Used to assign values to variables.
  • Examples: = (assign), += (add and assign), -= (subtract and assign).
  • Example: let x = 10; (The = operator assigns the value 10 to x.)

    • 5. Bitwise Operators:- Perform operations on the binary representations of numbers.
  • Examples: & (AND), | (OR), ^ (XOR), ~ (NOT), << (left shift), >> (right shift).
  • Example: let result = 5 & 1; (The & operator performs a bitwise AND on 5 and 1.)

    • 6. String Operators:- Used to concatenate strings.
  • Example: + (concatenation).
  • Example: let fullName = firstName + " " + lastName; (The + operator concatenates firstName and lastName.)

    • 7. Conditional Operators:- A shorthand for 'if-else' statements.
  • Example: condition ? expr1 : expr2.
  • Example: let canVote = age >= 18 ? "Yes" : "No"; (Checks if age is 18 or more.)

    • Q3. Differentiate between unary, binary, and ternary operators in JavaScript. Give examples of each.

    1. Unary Operators:- Unary operators operate on a single operand.
  • Unary Plus('+') :- Converts its operand to a number.
  • Example: 'let num = +"5"; // num becomes 5 (number)'

  • Unary Minus (-): Negates the value of its operand
  • Example: let neg = -10; // neg becomes -10'

  • Increment (++): Increases its operand by 1
  • Example: let x = 5; x++; // x becomes 6

  • Decrement (--): Decreases its operand by 1.
  • Example: let y = 5; y--; // y becomes 4

  • Logical NOT (!): Inverts the boolean value of its operand.
  • Example: let isTrue = true; let isFalse = !isTrue; // isFalse becomes false

  • Typeof (typeof): Returns the type of the operand.
  • Example: typeof 42; // returns "number"

    • 2. Binary Operators:- Binary operators require two operands (two values) and perform operations on them.
  • Arithmetic Operators: Perform mathematical calculations.
  • Example: let sum = 10 + 5; // sum becomes 15

  • Comparison Operators: Compare two values.
  • Example: let isEqual = 10 == "10"; // isEqual becomes true

  • Logical Operators: Combine or invert boolean values.
  • Example: let result = true && false; // result becomes false

  • Assignment Operators: Assign values to variables.
  • Example: let x = 5; x += 3; // x becomes 8

  • Bitwise Operators: Perform operations at the binary level.
  • Example: let result = 5 & 1; // result becomes 1

    • 3. Ternary Operator:- The ternary operator is a conditional operator that requires three operands. It is a shorthand way of writing an if-else statement.
  • Syntax: condition ? expr1 : expr2
  • Example: let age = 18;
    let canVote = age >= 18 ? "Yes" : "No"; // canVote becomes "Yes"

    • Q4. Discuss the precedence and associativity of operators in JavaScript. Why is understanding these concepts important?

    Operator precedence determines the order in which operations are performed in an expression with multiple operators. Operators with higher precedence are evaluated before those with lower precedence.
    Operator associativity determines the order in which operators of the same precedence level are evaluated. Associativity can be either:
  • Left-to-right (left associative): Operators are evaluated from left to right.
  • Right-to-left (right associative): Operators are evaluated from right to left.

  • Why Understanding Precedence and Associativity is Important:
    • 1. Avoiding Logical Errors: Misunderstanding operator precedence and associativity can lead to unexpected results in expressions, causing logical errors in your code.
    Example: let result = 5 + 2 * 10; (If you expect 70 but get 25 instead).
    2.Writing Clear Code: Knowing these concepts allows you to write more concise code without unnecessary parentheses, making your code easier to read and understand.
    3.Debugging and Maintenance: When you encounter a bug or need to modify existing code, understanding how expressions are evaluated helps you quickly identify and correct issues.
    4.Performance Considerations: Properly structured expressions can improve performance by minimizing unnecessary computations.
